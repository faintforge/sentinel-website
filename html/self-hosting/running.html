Running Sentinel

Starting the Sentinel service

In order to have Sentinel run on machine start and have it run in the background, a systemd service is needed.

A sentinel.service file is located inside the sentinel directory. Move it to /etc/systemd/system/.

$ sudo cp sentinel.service /etc/systemd/system/

Afterwards, the systemd daemon needs to be reloaded and the system needs to be enabled and started.

$ sudo systemctl daemon-reload
$ sudo systemctl enable --now sentinel

Running NGINX

Now a gunicorn instance is running Sentinel exposing a WSGI socket. NGINX can now be started.

$ sudo systemctl enable --now nginx

This should be everything needed to install and run the Sentinel password manager. Go to your website and revel in the glory of being secure.

If any problems have cropped up during this process, visit the FAQ section on GitHub.
